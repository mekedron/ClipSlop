# Prompt Assistant

```
⌃⌘⌥P → "the grammar prompt doesn't handle passive voice — fix it" → Approve
```

The Prompt Assistant is a floating, always-on-top chat window where an AI assistant edits your **prompt library** for you. Instead of digging through **Settings → Prompts**, you describe what you want in plain language and the assistant finds the right prompt, folder, or setting and proposes the change.

Every change is shown as an **Approve / Reject** card and applied only after you confirm — nothing is modified without your explicit approval.

## Opening it

Two ways in:

- **Global shortcut** — <kbd>⌃⌘⌥P</kbd> by default. Reconfigurable in **Settings → General → Keyboard Shortcuts**.
- **Menu-bar menu** — click the ClipSlop menu-bar icon and pick **Prompt Assistant**.

The window is a frosted, always-on-top panel, so it stays in view while you keep working in another app.

## How it works

1. **Ask in plain language.** For example: *"the grammar prompt doesn't handle passive voice — fix it"*, or *"make a Legal folder and move the contract prompts into it"*.
2. **The assistant proposes a change.** It finds the relevant prompt, folder, or setting and drafts the edit.
3. **You approve or reject.** Each proposed change appears as an Approve / Reject card. It's applied only when you confirm.

<Callout type="warn" title="Deletes ask twice">
  Deleting a prompt or folder shows an extra warning before it happens. Deleting a non-empty folder also shows how many items (its descendant count) would go with it, so you don't lose more than you meant to.
</Callout>

## What it can do

The assistant works through tool calls, so it can touch anything the prompt library exposes:

- **Prompts** — create, edit, and delete prompts.
- **Folders** — create, rename, and delete folders.
- **Organisation** — move prompts between folders.
- **Global shortcuts** — assign or clear either per-prompt slot: [Inline Run](../use/quick-paste.mdx) (transform the selection in place) and [Run in Editor](../use/open-and-run.mdx) (open the popup and run).
- **Display mode** — set a prompt's display mode: Plain text, HTML, Markdown, or Markdown Styled.
- **Capture option** — toggle a prompt's "select all before capture" behaviour.
- **Per-prompt provider** — set the provider override for a prompt.

<Callout type="note" title="Every edit still lands in the library">
  The Prompt Assistant is a faster front end for the same [Prompt library](./prompt-library.mdx) you edit by hand in **Settings → Prompts**. Anything it changes shows up there, and anything you change there is visible to the assistant.
</Callout>

## Chatting in the window

The chat input reuses the same field as the <kbd>⌘K</kbd> quick-instruction bar, so it behaves the same way:

| Key | Action |
|-----|--------|
| <kbd>Enter</kbd> | Send the message |
| <kbd>⇧Enter</kbd> | New line |
| <kbd>Esc</kbd> | Close the window |

## Choosing a provider

A small provider switcher in the window's header lets you pick which tool-calling provider the assistant uses, and jump straight to **Provider Settings**.

The Prompt Assistant relies on **AI tool calling**, so it works with any provider that supports it:

- OpenAI (API key)
- OpenAI (Sign In / ChatGPT)
- Anthropic
- Ollama
- OpenAI-compatible endpoints

<Callout type="warn" title="Not available on the CLI Tool provider">
  The **CLI Tool** provider does not support tool calling. When it's selected, the Prompt Assistant shows a notice asking you to switch to a tool-calling provider — use the header's provider switcher to pick one.
</Callout>

## Localisation

The Prompt Assistant is fully localised into all 17 supported languages, and the onboarding flow ends with a **Prompt Assistant** step whose **Try it** button opens the window and auto-sends a localised greeting to get you started.
