# Install &amp; first run

Three steps: install, open past the Gatekeeper warning, grant permissions. About three minutes total.

## Install

### Via Homebrew

```bash
brew tap mekedron/tap
brew install --cask clipslop
```

### Via the DMG

Grab the latest `.dmg` from [Releases](https://github.com/mekedron/ClipSlop/releases/latest) and drag `ClipSlop.app` to your **Applications** folder.

After install, ClipSlop lives in your menu bar:

## Opening the app for the first time

Drag `ClipSlop.app` to your **Applications** folder and double-click. It opens straight away — ClipSlop is signed with an Apple Developer ID and notarised by Apple, so there's no Gatekeeper warning to click through.

The app is safe, and you can verify that independently: the source is fully open, and every release is built and signed by [GitHub Actions](https://github.com/mekedron/ClipSlop/actions) directly from the public repository. To check the copy you downloaded:

```bash
spctl -a -vvv -t exec /Applications/ClipSlop.app
# ClipSlop.app: accepted
# source=Notarized Developer ID
```

## Grant permissions

ClipSlop needs two macOS permissions to work in any app:

- **Accessibility** — required to read the text you've selected and paste results back. Grant in **System Settings → Privacy &amp; Security → Accessibility**.
- **Screen Recording** — only needed for the OCR feature (`⇧⌘2`). Grant in **System Settings → Privacy &amp; Security → Screen Recording**.

## After updating to a new version

Nothing to do. Because releases are signed with a stable Developer ID, macOS keeps your Accessibility and Screen Recording grants across updates.

<Callout type="warn">
**One exception — upgrading from v1.x.** Releases before v2.0.0 were unsigned, and macOS ties permissions to the app's signature. Moving from an unsigned build to a signed one invalidates the old grant once, so if shortcuts stop working right after that upgrade:

1. Open **System Settings → Privacy &amp; Security → Accessibility**.
2. Find ClipSlop in the list and **remove it** (select → click "−").
3. Click "+" and **re-add** `ClipSlop.app` from your Applications folder.
4. Repeat for **Screen Recording** if you use OCR.

This is a one-time migration, not a per-update chore. Your prompts and settings are never affected.
</Callout>
